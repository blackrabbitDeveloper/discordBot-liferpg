# Life RPG Discord Bot MVP 설계서

## 1. 문서 목적

이 문서는 **“인생을 게임처럼 운영하는 Discord Bot MVP”**를 설계하기 위한 기준 문서다.  
목표는 완성형 서비스 제작이 아니라, 다음 핵심 가설을 빠르게 검증하는 것이다.

1. 사용자는 디스코드 환경에서도 하루 행동을 “퀘스트”처럼 받아들이는가
2. 자동 추천된 일일 퀘스트를 실제로 선택하고 수행하는가
3. 자동 리포트와 약한 게임화만으로도 3일 이상 반복 사용이 가능한가
4. 소규모 길드/파티 구조가 유지율에 긍정적인 영향을 주는가

---

## 2. 제품 한 줄 정의

**사용자의 현실 행동을 일일 퀘스트, 성장, 리포트, 협동 구조로 변환해 삶을 게임처럼 운영하게 만드는 Discord Bot**

---

## 3. MVP 목표

### 3.1 핵심 목표
- `/start` 한 번으로 온보딩과 첫 퀘스트 경험 제공
- 이후에는 자동 메시지 중심으로 일일 루프 운영
- 유저는 복잡한 명령어 없이 버튼 중심으로 참여
- 하루 퀘스트, 미완료 처리, 일일 리포트, 주간 리포트까지 자동화

### 3.2 검증 지표
- 온보딩 완료율
- 첫 퀘스트 선택률
- 첫날 퀘스트 완료율
- Day2 유지율
- Day3 유지율
- 주간 리포트 열람률
- 길드 참여 유저 vs 비참여 유저 유지율 비교

---

## 4. MVP 핵심 원칙

### 4.1 명령어 최소화
MVP는 “명령어 기반 봇”이 아니라 **자동으로 굴러가는 개인 게임 매니저**처럼 보여야 한다.

### 4.2 퀘스트는 고정 콘텐츠가 아니라 반자동 제안
퀘스트를 수백 개 만들어두는 방식이 아니라,
- 사용자의 목표
- 오늘의 시간/에너지
- 최근 성공률
을 바탕으로 오늘 가능한 선택지를 제안해야 한다.

### 4.3 실패는 패널티보다 상태 전환
하루 퀘스트를 못 했다고 강한 실패 처리나 죄책감 유도 문구를 쓰면 안 된다.  
미완료는 “기록”으로 남기고, 오늘은 새 턴으로 넘어간다.

### 4.4 과거 퀘스트는 회고 대상
지난 날짜 퀘스트 버튼을 눌렀을 때는 정상 보상을 주지 않는다.  
대신 “회고 기록”으로만 남기거나 축소 보상을 주는 방식으로 처리한다.

### 4.5 길드는 경쟁이 아니라 유지력 보조 장치
랭킹 중심이 아니라,  
- 같이 하고 있다는 감각
- 가벼운 응원
- 주간 공동 목표
를 제공하는 구조여야 한다.

## 5. 서버 구조 제안

### 5.1 서버 이름 예시
- Life RPG Lab
- Habit Game Test
- 루틴 실험실

### 5.2 채널 구성
- `#공지`
- `#사용방법`
- `#오늘-플레이`
- `#피드백`
- `#자유채팅`

### 5.3 권장 운영 방식
- 봇 공지 및 자동 메시지는 `#오늘-플레이` 또는 개인 DM 사용
- 길드 기능을 붙일 경우 길드별 전용 채널 또는 스레드 생성 가능
- 초기 테스트 인원은 10~30명 규모 권장

---

## 6. 사용자 여정

### 6.1 첫 진입
1. 사용자가 서버에 입장
2. `#사용방법` 확인
3. `/start` 실행
4. 온보딩 질문 응답
5. 첫 퀘스트 3개 추천
6. 하나 선택
7. 완료 처리
8. 첫 보상 지급

### 6.2 일상 사용
1. 아침 자동 메시지 수신
2. 오늘 퀘스트 3개 확인
3. 버튼으로 선택
4. 완료 또는 건너뜀 처리
5. 저녁 자동 리포트 수신

### 6.3 주간 사용
1. 주간 요약 자동 수신
2. 가장 성장한 영역 확인
3. 다음 주 추천 전략 확인
4. 길드 공동 목표 달성 여부 확인

---

## 7. 명령어 설계

### 7.1 필수 명령어
#### `/start`
온보딩 시작 + 첫 퀘스트 추천까지 연결

#### `/status`
현재 상태 확인
- 레벨
- 경험치
- 스트릭
- 스탯
- 현재 목표

#### `/goal`
목표 수정

#### `/pause`
쉬기 모드 전환
- 알림 빈도 낮춤
- 가벼운 회복 퀘스트 위주 제안

### 7.2 MVP에서 제외 가능한 명령어
- `/today`
- `/choose`
- `/done`
- `/weekly`
- `/guild`

이 기능들은 가능하면 **자동 메시지 + 버튼 인터랙션**으로 대체한다.

## 8. 온보딩 설계 (`/start`)

### 8.1 온보딩 목표
- 3분 이내 완료
- 첫 퀘스트까지 도달
- 첫 보상 경험까지 이어지게 구성

### 8.2 질문 흐름
#### Step 1. 가장 바꾸고 싶은 영역
버튼 예시:
- 건강
- 집중
- 커리어/일
- 공부
- 창작
- 돈관리
- 생활 리듬

#### Step 2. 현재 주요 목표
자유 입력 또는 선택형
예:
- 매일 10분 운동
- 게임 기획안 작성
- 웹소설 집필
- 가계부 기록
- 수면 패턴 정상화

#### Step 3. 하루 여유 시간
- 10분 이하
- 10~30분
- 30분 이상

#### Step 4. 현재 에너지 상태
- 낮음
- 보통
- 높음

#### Step 5. 플레이 성향
- 아주 가볍게
- 적당히
- 조금 빡세게

### 8.3 첫 퀘스트 추천 예시
```
오늘의 첫 퀘스트를 준비했어요

1. 5분 정리
2. 목표 관련 10분 행동
3. 물 한잔 + 짧은 스트레칭

이 중 하나만 해도 첫 경험치를 얻을 수 있어요
```

### 8.4 첫 완료 보상
- XP 지급
- 스탯 1~2 상승
- 스트릭 시작
- 짧은 성공 메시지 출력

## 9. 일일 루프 설계

### 9.1 아침 자동 메시지
발송 시간은 기본 오전 8시~10시 중 랜덤 또는 설정값 기반

예시:
```
오늘의 퀘스트가 도착했어요

현재 상태를 기준으로 3개를 골라봤어요
- 5분 정리
- 목표 관련 10분 행동
- 가벼운 회복 루틴

원하는 흐름을 선택해 주세요
```

버튼:
- 이걸 할래요
- 더 쉬운 걸로
- 오늘은 회복 모드
- 오늘은 쉬어갈래요

### 9.2 퀘스트 선택 후
예시:
```
좋아요. 오늘의 선택:
[목표 관련 10분 행동]

완료하면:
+10 XP
집중 +2
스트릭 유지
```

버튼:
- 시작하기
- 완료했어요
- 못 했어요
- 다른 걸로 바꿀래요

### 9.3 완료 처리
예시:
```
퀘스트 완료!

+10 XP
집중 +2
스트릭 3일 유지 중
좋은 흐름이에요
```

### 9.4 미완료 처리
예시:
```
괜찮아요.
오늘은 흐름이 조금 무거웠나 봐요.

내일은 더 가벼운 퀘스트로 다시 시작할 수 있게 준비할게요.
```

### 9.5 건너뜀 처리
예시:
```
오늘은 쉬어가는 턴으로 기록할게요.
기록은 남지만, 괜찮아요.
내일 다시 이어가면 됩니다.
```

## 10. 퀘스트 상태 설계

각 퀘스트는 다음 상태를 가진다.

- `PENDING` : 제안됨 / 아직 처리 안 됨
- `COMPLETED` : 오늘 완료
- `SKIPPED` : 사용자가 명시적으로 건너뜀
- `EXPIRED` : 날짜가 지나 자동 종료
- `LATE_LOGGED` : 날짜는 지났지만 회고 기록으로 남김

### 10.1 하루가 지나면
전날의 `PENDING` 퀘스트는 자동으로 `EXPIRED` 처리한다.  
삭제하지 않고 기록으로 남긴다.

### 10.2 과거 퀘스트 버튼 클릭 시
정상 완료 처리하지 않는다.

예시 응답:
```
이 퀘스트는 어제의 기록이에요 👀

지금 완료 처리하면
보상은 지급되지 않고 회고 기록으로만 남아요.

그래도 기록할까요?
```

버튼:
- 기록만 하기
- 오늘 퀘스트 보기

### 10.3 회고 기록 처리
- 상태: `LATE_LOGGED`
- 보상: 없음 또는 매우 소량
- 분석 데이터에는 반영 가능

---

## 11. 스트릭 설계

### 11.1 기본 원칙
하루 미완료 시 바로 리셋하면 이탈 위험이 높다.  
MVP에서는 완만한 구조를 권장한다.

### 11.2 추천 방식
- 당일 1개 이상 완료 시 스트릭 유지
- 1일 미완료: 스트릭 보류 또는 보호
- 2일 연속 미완료: 스트릭 감소
- 3일 연속 미완료: 스트릭 끊김

### 11.3 스트릭 관련 UX 문구
좋은 예:
- 어제는 쉬어간 날이었어요
- 오늘 다시 이어갈 수 있어요
- 흐름을 회복할 기회예요

피해야 할 예:
- 실패했습니다
- 스트릭이 깨졌습니다
- 오늘도 안 하면 손해입니다

---

## 12. 리포트 설계

### 12.1 즉시 리포트
퀘스트 완료 직후 짧게 제공
- XP
- 스탯
- 스트릭 상태
- 길드 기여

### 12.2 일일 리포트
저녁 자동 발송

예시:
```
오늘 결과예요

- 완료: 2개
- 가장 많이 성장한 영역: 집중
- 스트릭: 4일
- 오늘의 흐름: 안정적

내일은 비슷한 강도로 이어갈게요
```

### 12.3 주간 리포트
일요일 저녁 발송 권장

예시:
```
이번 주 요약

- 완료율: 68%
- 가장 많이 성장한 영역: 실행력
- 어려웠던 패턴: 금요일 저녁
- 추천 전략: 10분 이하 퀘스트 비중 확대

다음 주에는 조금 더 가볍게 이어가는 걸 추천해요
```

## 13. 퀘스트 추천 로직

### 13.1 초기 버전
AI 없이 룰 기반으로 충분

### 13.2 입력값
- 목표 카테고리
- 하루 여유 시간
- 현재 에너지
- 플레이 강도 선호
- 최근 3일 완료율
- 최근 실패/건너뜀 횟수

### 13.3 출력값
- 오늘의 추천 퀘스트 3개
- 예상 소요 시간
- 예상 보상
- 추천 이유

### 13.4 로직 예시
#### 건강 목표 + 에너지 낮음 + 시간 적음
- 물 마시기 + 가벼운 스트레칭
- 5분 걷기
- 수면 준비 체크

#### 창작 목표 + 에너지 보통 + 시간 보통
- 아이디어 3줄 정리
- 초안 10분 쓰기
- 기존 문서 5분 수정

#### 최근 실패율 높음
- 난이도 자동 완화
- 5분 이하 퀘스트 비중 증가
- “회복 모드” 추천 빈도 증가

---

## 14. 퀘스트 카테고리

초기에는 아래 공통 카테고리만 있으면 충분하다.

- 건강
- 집중
- 일/커리어
- 공부
- 창작
- 돈관리
- 정리/생활
- 회복

### 14.1 중요한 점
퀘스트를 계속 “해금”하는 느낌보다,
**추천되는 선택지의 깊이와 난이도가 넓어지는 구조**가 더 적합하다.

예:
- 창작(가벼움): 아이디어 3줄
- 창작(보통): 초안 10분
- 창작(깊음): 25분 집중 작업

## 15. 길드 / 파티 설계 (선택 기능)

### 15.1 MVP 반영 여부
핵심 루프 검증이 먼저다.  
길드는 MVP 1차에서는 제외 가능하며, 유지율이 확인되면 2차에 추가한다.

### 15.2 길드 기본 원칙
- 경쟁보다 협동
- 실시간보다 비동기
- 채팅보다 가벼운 상호작용
- 부담보다 유지력 보조

### 15.3 권장 길드 규모
- 3~8명
- 최대 10명 이하

### 15.4 기능 예시
- 길드원 누군가 퀘스트 완료 시 진행도 상승
- 주간 공동 목표 달성 시 길드 보너스
- 이모지 응원
- 스트릭 회복 응원 메시지

### 15.5 피해야 할 기능
- 전체 순위
- 개인 순위
- 강제 참여 압박
- 복잡한 실시간 채팅 미션

## 23. 핵심 요약

이 MVP의 본질은 “게임 같은 할 일 봇”이 아니다.

이 봇은:
- 사용자의 하루를 작은 턴으로 나누고
- 행동을 퀘스트로 제안하며
- 실패를 부드럽게 흡수하고
- 성장과 회고를 자동으로 보여주는
**개인 행동 운영 시스템**이다.

핵심은 화려한 시스템이 아니라 아래 루프가 살아 있는지다.

**온보딩 → 오늘 퀘스트 제안 → 행동 → 완료/미완료 처리 → 리포트 → 다음 날 재진입**

이 루프가 유지되면, 이후 웹/앱으로 확장할 가치가 있다.
